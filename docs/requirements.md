# 需求分析文档

## 1. 项目定位

VideoMind 是一个 **AI 视频学习助手**，帮助用户将视频内容转化为结构化知识。通过 AI 自动总结、生成学习笔记和思维导图，让用户高效理解视频内容，而非单纯下载存储。

## 2. 目标用户

| 用户类型 | 核心需求 |
|----------|----------|
| 技术学习者 | 观看教程视频后，AI 生成结构化笔记用于复习 |
| 知识管理者 | 整理视频知识库，随时回顾关键概念 |
| 内容研究者 | 快速了解视频核心内容，无需完整观看 |
| 普通用户 | 保存/下载视频到本地 |

## 3. 核心需求

### V1.0（已完成）

| 优先级 | 需求 | 说明 |
|--------|------|------|
| P0 | 多平台视频链接解析 | 粘贴链接即可获取视频信息（标题、时长、封面、格式列表） |
| P0 | 多清晰度选择 | 支持多清晰度选择 |
| P0 | 视频下载 | 选择格式后一键下载 |
| P0 | 实时下载进度 | WebSocket 推送下载进度 |

### V2.0（当前版本 — AI 学习）

| 优先级 | 需求 | 说明 |
|--------|------|------|
| P0 | AI 智能总结 | 输入视频 URL，AI 流式生成内容摘要 |
| P0 | 字幕提取 | 优先 B 站 CC 字幕 API，降级 yt-dlp，支持 SRT/VTT/TXT 格式 |
| P0 | 结构化学习笔记 | AI 自动生成标题层级分明的 Markdown 笔记，支持代码块、重点高亮 |
| P1 | 思维导图 | 将总结内容可视化为交互式思维导图（markmap），支持 SVG/PNG 导出 |
| P1 | AI 问答 | 基于字幕内容的多轮流式对话 |
| P1 | Markdown 渲染 | AI 输出内容结构化展示（摘要、笔记、聊天消息） |
| P1 | 渐进式生成 UI | 显示 AI 处理各阶段状态（字幕加载 → 摘要生成 → 笔记生成 → 导图生成） |
| P1 | 移动端响应式 | 768px 断点适配：输入框垂直堆叠、Tab 栏缩小、内容区 vh 限制、触控目标 ≥44px |
| P1 | 学习历史 | 自动保存 AI 分析结果，支持搜索、标签过滤、平台筛选、收藏、删除 |
| P1 | 语义搜索 | 基于 ChromaDB 的跨视频语义搜索 |
| P1 | 用户认证 | Session Cookie 登录注册 + 游客模式，三级权限隔离 |

## 4. 支持平台

重点支持：B站 · YouTube · 抖音 · 小红书 · TikTok

## 5. 非功能性需求

- **易用性**：粘贴链接即可解析，最少操作步骤
- **渐进式反馈**：AI 处理各阶段均有 UI 反馈，避免长时间等待
- **性能**：解析时间 < 3s，AI 流式首 token < 5s
- **轻量化**：SQLite + ChromaDB，无需重型数据库
- **兼容性**：Windows/macOS/Linux 均可运行，主流浏览器支持

## 6. 约束条件

- 使用 yt-dlp 作为核心视频引擎
- 仅供学习使用，尊重版权
- 前端采用 **VideoMind 品牌风格**：深蓝黑渐变背景（#0F172A）、蓝青渐变主色（#3B82F6 → #06B6D4）、毛玻璃卡片、简洁现代的暗色主题
- 目标用户以国内为主，平台展示和文案国内优先
- 服务器配置：4C4G，50G 存储，适合个人使用
